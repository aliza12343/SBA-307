Architecture

Key Design Decisions

Every choice has a reason

JWT over Sessions

Stateless tokens scale horizontally — no session store needed. The frontend decodes the JWT to check expiry locally before making API calls.

useReducer over useState

Auth and cart state have multiple interdependent transitions. useReducer makes state machines explicit and prevents inconsistent intermediate states.

Centralized API Client

All fetch calls go through a single apiClient.js that injects Bearer tokens and enforces a 10-second AbortController timeout — no duplication.

Externalized Secrets

${ENV_VAR:default} in application.properties means local dev works without environment setup, while prod reads from AWS SSM — same codebase.

Technical Deep Dive

Security

Three vulnerabilities found and fixed

Privilege Escalation

✗ Before: POST /api/users/register accepted "role":"ADMIN" in the request body — anyone could make themselves admin.

✓ Fixed: Removed the endpoint. The secure /api/auth/register always assigns USER role. Admin-only user management is behind ADMIN role guard.

Silent Token Pass-Through

✗ Before: JwtAuthFilter called chain.doFilter() even when a token was present but invalid — attacker could use expired tokens.

✓ Fixed: Returns HTTP 401 with JSON body immediately when token fails validation.

Hardcoded JWT Secret

✗ Before: Secret was a static final string in JwtUtil — impossible to rotate without redeploying.

✓ Fixed: Constructor injection via @Value("${jwt.secret}") reads from env var in prod, sensible default locally.

Rate Limiting

✓ Added: Bucket4j RateLimitFilter throttles /api/auth/login and /api/auth/register to 10 requests/minute per IP. Returns HTTP 429 on breach.

Technical Deep Dive

React Patterns

Correct hooks, every time

Rules of Hooks Bug Found

Early returns were placed BEFORE useState calls — this crashes React at runtime. All hooks must be declared unconditionally before any conditional returns.

JWT Expiry on Load

AuthContext decodes the JWT payload with atob() and compares exp * 1000 to Date.now() before restoring auth state from localStorage.

// WRONG — hook after conditional return
const { user } = useAuth()
if (user) return <Navigate />
const [form, setForm] = useState({}) // ← crash

// CORRECT — all hooks first
const { user } = useAuth()
const [form, setForm] = useState({})  // ← safe
const [errors, setErrors] = useState({})
if (user) return <Navigate />  // ← after hooks

AbortController Timeout

Every API call has a 10-second timeout via AbortController. Aborted fetches throw 'Request timed out.' to show the user a clear message.

AWS Infrastructure

Cloud Deployment

Built cloud-ready from day one

Compute — EC2

Spring Boot JAR deployed on EC2. Port 8081 behind a Security Group. Systemd service for auto-restart on reboot.

Database — RDS MySQL

Managed MySQL 8 in a private subnet. Hibernate auto-runs DDL on startup. Connection string via SSM Parameter Store.

Frontend — S3 + CloudFront

vite build produces a static bundle. S3 bucket + CloudFront distribution serves it globally with HTTPS.

Secrets — SSM Parameter Store

Three parameters: DB_URL, DB_PASSWORD, JWT_SECRET. Spring reads via ${ENV_VAR:local-default} — same JAR, any environment.

CORS

Allowed origins come from CORS_ORIGINS env var — CloudFront URL in prod, localhost in dev. No code change needed between environments.

Swagger UI

API docs at /swagger-ui.html — auto-generated from controller annotations. Useful for peer review and integration testing.

Lessons Learned

Reflection

What I would do differently

Challenges Overcome

CORS — three different configs

Three controllers had @CrossOrigin("*") while WebConfig did nothing. Consolidated into one WebMvcConfigurer — single source of truth.

React hook ordering bug

Found by tracing a runtime crash. The fix was strict: all hooks are declared before any conditional returns, no exceptions.

JWT secret in source control

Caught during security review. Moved to @Value injection so the secret lives in env vars, not the repo.

If Starting Over

Refresh Tokens from day one

Short-lived access tokens with a separate refresh token flow would eliminate the need for users to log in again when the JWT expires.

Test-first on the backend

Writing unit tests after the services were done caught bugs, but TDD would have caught the privilege escalation vulnerability before it was ever committed.

Pagination on product lists

The shop currently loads all products. Pageable + Page<T> from Spring Data JPA would be a one-line change — and essential at scale.
